How to add a Mods resource pack:

You need to have an GitHub account first
You need to verify if the Mod doesn't already have a repository

  1. Create a repository in your own profile, the repository need to have the same name as the assets folder file (ex: a Actually Additions mod repository would be named: actuallyadditions).

  2. Remove the master branch & add a branch for each supported version, set default branch to the highest supported version.

  3. Add your textures, use the same pack.png as the Faithful Mods Logo. You also need to add a pack.mcmeta following this setup:

    					
					{
						"pack":{
						"pack_format": 3,
						"description": "Faithful Mods"
						}
					}

    The "pack_format" can be found here (make a pull request if outdated).

  4. Your repository should look like this:

  5. Then make a pull request modifying this file following this setup:

              
					[
							{
									... already existing mod ...
							},
							{
									"name": [
											"Actually Additions",     // Name with capital & spaces
											"actuallyadditions"       // Name from the `assets` file
									],
									"versions": [                 // Add supported versions
											"1.12.2",
											"1.10.2"
									],

									// Get the repository owner, can be your username or the organization name
									// You can find it in the url of your repository: https://github.com/Faithful-Mods/actuallyadditions basically your organization or username

									"repository": "Faithful-Mods"
							},
							{
									... already existing mod ...
							}
					]

    Please add it following an alphabetical order

  6. Final step: if your pull request is merged, you will also received an invitation to be a member of the Faithful Mods Organization. From this, you will be able to move your repository inside our Organization (don't forget to modify the mods.json file with the new repository owner: replace your username with Faithful-Mods).
